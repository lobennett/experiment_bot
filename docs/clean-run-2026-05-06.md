## What changed in the framework before this clean run

The pilot integration was added as Stage 6 of the Reasoner during this
session. Three pilot-side gaps surfaced and were closed before all four
paradigms passed cleanly:

1. **Splice was full-replace** (commit `5b3da40`). When the LLM's
   refinement returned a partial `runtime` block, the splice clobbered
   sibling sub-fields (e.g. lost `advance_behavior`). Replaced with a
   recursive deep-merge that preserves untouched sub-fields and only
   overlays the changed ones.

2. **`max_blocks` was a hard stop, not advisory** (this commit's
   parent). Pilot exited at the first FEEDBACK phase by default, which
   on paradigms with trial-by-trial feedback meant exiting after 1–3
   trials, far below `min_trials`. Replaced with: pilot always
   advances past feedback; stops are bounded only by `min_trials` +
   target conditions, wall-clock timeout, or consecutive-misses cap.

3. **Pilot crashed on withhold-response sentinels.** When a
   stop-signal trial sampled `response_key="withhold"` (the executor's
   sentinel for "no keypress"), pilot tried to press the literal string
   `"withhold"` and Playwright raised. Pilot now reuses
   `TaskExecutor._is_withhold_sentinel()` so withhold trials skip the
   keypress entirely.

These three gaps are pilot-mechanics issues, not paradigm-specific
prompt edits. None of them encode jsPsych assumptions or
cognitive-control priors; all of them generalize to any future paradigm.

After the batch completes, run the oracle per paradigm class:

```bash
uv run experiment-bot-validate --paradigm-class conflict --label expfactory_stroop
uv run experiment-bot-validate --paradigm-class conflict --label cognitionrun_stroop
uv run experiment-bot-validate --paradigm-class interrupt --label expfactory_stop_signal

```

Each produces a JSON report under `validation/<label>_<timestamp>.json`
with per-pillar (rt_distribution, sequential, individual_differences)
pass/fail against published meta-analytic ranges from
`norms/conflict.json` / `norms/interrupt.json`.
